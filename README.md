# computingpower-service
使用原生的技术框架，形成网页原型系统，不需要后端代码，可以使用模拟数据，实现业务模块的交互与逻辑闭环；

系统一：算力资源调度管理系统（核心运营软件）

核心作用：统筹所有算力节点（服务器、GPU、存储、网络），实现资源分配、任务调度、负载均衡，保障算力租赁高效运转，避免资源闲置。
•	算力节点全景管理：实时展示所有服务器、GPU、存储设备的在线状态、负载情况、资源占用率，支持按机房、机柜、设备型号筛选，快速定位闲置/满载资源。
•	客户算力分配模块：根据客户租赁需求（算力规格、使用时长、并发量），灵活分配GPU节点、服务器资源，支持按算力卡数、算力性能（TFLOPS）精准分配，分配后实时同步客户端。
•	任务调度与优先级管理：接收客户训练/推理任务，自动调度空闲算力资源，支持任务优先级设置（付费越高优先级越高），避免任务排队拥堵，优化算力利用率。
•	资源弹性扩容/缩容：根据客户需求变化，快速调整算力分配（如临时增加GPU卡数、扩展存储），支持按小时/天弹性调整，同步更新计费信息，适配训推一体、短期算力峰值需求。
•	算力负载均衡：实时监测各节点负载，自动将高负载任务分配至空闲节点，避免单节点过载、掉卡，保障客户任务稳定运行，提升整体算力利用率（目标≥85%）。
•	算力节点健康诊断：定期检测所有算力节点硬件状态（GPU温度、CPU负载、内存占用），发现异常自动预警，标记故障节点并暂停分配任务，联动运维系统处理。
•	资源使用统计分析：统计各节点、各客户的算力使用时长、资源占用量，生成日报/周报/月报，为运营决策（资源扩容、定价调整）提供数据支撑。
•	算力预留管理：支持客户提前预留算力资源（如长期租赁客户锁定固定节点），预留期限可设置，到期自动释放，避免资源浪费，保障长期客户权益。

任务：完成系统页面的HTML+CSS+JS编码，实现交互流程，填充模拟数据。不需要实现后端。
